# 校园噪音分贝预警员系统 — 3 分钟演示剧本（优化版）

> **时长**: ≤3 分钟（约 180 秒）
> **要求**: 真人语音解说，可看稿；重可视化演示，不讲代码
> **核心卖点**: 混合动态阈值引擎 92.6% 准确率 · ccswitch 智能化 API 实时调整 · 三重复合触发条件自动切换

---

## 准备工作（录制前必做）

1. 双击 `一键启动前后端.bat`，确认三个服务就绪：
   - 前端 `http://localhost:5173`
   - 后端 `http://localhost:8080`
   - ccswitch `http://localhost:5000`（如果没启也不影响主流程）
2. 浏览器打开 `http://localhost:5173`，确认仪表盘有数据
3. 提前登录管理员 `admin / admin123`，让页面停留在仪表盘
4. 关闭无关标签页、通知弹窗
5. 打开本稿子放在屏幕旁边，测试麦克风音量

## 详细剧本

### ① 开场定调 + 仪表盘全景（0:00 - 0:20）— 20 秒

**画面**: 镜头给仪表盘页面全景，从顶部的绿色混合引擎面板一直看到下面四个功能区卡片

**解说词**:

> 大家好，我们做的是校园噪音分贝预警员系统。它覆盖图书馆、食堂、操场、宿舍四大功能区，核心是一套会自己根据数据动态调整阈值的智能判断引擎——不是写死的 55 分贝一刀切，而是系统实时计算该用什么阈值来判断异常。最终准确率 92.6%，误报率只有 4%。下面用三分钟演示一下。

**操作**:
- 0:00-0:10 鼠标从页面顶部缓慢往下滑，展示仪表盘全局
- 0:10-0:15 鼠标指向顶部绿色面板的 `ccswitch 混合动态阈值引擎` 标题
- 0:15-0:20 鼠标指向 `混合模型准确率 92.6% | 远超固定阈值 78.0%` 这个绿色徽章

> **设计意图**: 开门见山把"动态调整"和"92.6%"两个核心数字砸出来，让观众 20 秒内就知道这个系统的价值。


### ② 混合阈值引擎面板（0:20 - 0:55）— 35 秒

**画面**: 聚焦仪表盘顶部的混合阈值引擎面板，然后往下看四个功能区卡片

**解说词**:

> 看仪表盘顶部这个混合阈值引擎面板。左边标签显示当前判断模式——现在是"STAT_ADAPTIVE 统计自适应模式"，系统正在用滑动窗口实时计算动态阈值。右边四个卡片分别是：当前判断模式、触发原因、已判定记录总数、还有三种模型的判定分布——RULE 业务规则、ADAPT 统计自适应、HYBRID 混合模型各判了多少条，一目了然。

**操作**:
- 0:20-0:28 鼠标依次划过引擎面板的 4 个统计卡片（模式 → 触发原因 → 判定总量 → 模型分布）
- 0:28-0:32 鼠标指向下方四模型对比进度条（混合 92.6% 绿色 → 统计自适应 89.4% 蓝色 → 业务规则 88.7% 橙色 → 固定阈值 78% 红色）

**画面**: 镜头下移到四个功能区实时卡片

**解说词**:

> 下面是四个功能区的实时分贝卡片。每个卡片显示当前分贝值、当前阈值、还有状态指示灯——绿色表示正常，红色闪烁表示异常。注意看，同一个 60 分贝，在图书馆可能已经标红了，但在操场还是绿的——因为系统不是一刀切，它会根据每个功能区的噪声敏感度和当前时段，自动计算不同的阈值。数据每 10 秒自动刷新。

**操作**:
- 0:32-0:40 鼠标逐一划过四个功能区卡片（图书馆 → 食堂 → 操场 → 宿舍），每个停留 2 秒
- 0:40-0:45 鼠标指向某个卡片的阈值行，再指状态灯
- 0:45-0:50 鼠标指向右侧的 ccswitch 配置服务卡片（模型名、运行时长）
- 0:50-0:55 鼠标指向右侧的四模型对比迷你进度条

> **设计意图**: 这段是核心卖点展示——引擎面板的丰富数据 + 功能区卡片体现"同一分贝不同判定"的业务规则思想。用具体的 UI 元素让观众直观感受到"动态"和"智能"。


### ③ 阈值配置页 — 三种模型现场切换（0:55 - 1:35）— 40 秒

**画面**: 点击侧栏"阈值配置"，先展示功能区规则标签页

**解说词**:

> 点开阈值配置页面，这是系统的核心配置中心。先看功能区规则。以图书馆为例，不同时段有不同的阈值——早间时段可能只有 45 分贝，上午上课时段放宽到 50，夜间时段又收紧。每个功能区、每个时段，都是独立配置的。这就是"业务规则驱动"，阈值不是拍脑袋定的，是按实际场景来的。

**操作**:
- 0:55-0:58 点击侧栏"阈值配置"
- 0:58-1:05 鼠标指向表格中的时段列和阈值列，上下滑动展示不同行
- 1:05-1:08 切换到"食堂"标签页，展示不同的规则数值

**画面**: 切换到"自适应阈值"标签页

**解说词**:

> 切换到自适应阈值标签。这里四个功能区各有一张卡片——窗口大小、K 值、均值 μ、标准差 σ、动态上限、动态下限，全是实时计算出来的。比如图书馆，窗口 15 分钟、K 值 2，均值 48.3，标准差 3.2，动态上限就是 54.7。这个上限不是写死的，数据一变，它跟着变。这就是智能化。

**操作**:
- 1:08-1:10 点击"自适应阈值"标签
- 1:10-1:18 鼠标指向图书馆卡片，依次划过窗口大小→K值→均值→标准差→上限→下限
- 1:18-1:20 鼠标划过其他三个功能区的卡片

**画面**: 切换到"混合模型"标签页

**解说词**:

> 再看混合模型标签。这里能看到当前运行模式、是否触发了切换、还有 3 个滑动窗口的异常率进度条。下面是五个性能指标——准确率、精确率、召回率、F1 分数、误报率。最关键的是，当连续 3 个窗口异常率超过 10%、或者进入午休夜间等特殊时段、或者分贝骤升超过 15 分贝——三个条件满足任意一个，系统就会自动从统计自适应模式切换回更保守的业务规则模式。触发条件解除后再恢复。这就是它准确率能达到 92.6% 的原因。

**操作**:
- 1:20-1:22 点击"混合模型"标签
- 1:22-1:28 鼠标指向运行状态描述列表（当前模式标签、异常率进度条、触发原因）
- 1:28-1:35 鼠标逐一划过五个性能指标数字，最后停留在模式分布饼图上

> **设计意图**: 这是三段式展示三种模型的精髓——先看静态规则，再看动态参数，最后看混合切换逻辑。层层递进，让观众理解"为什么混合模型最好"。


### ④ 噪声监测 + 告警闭环（1:35 - 2:00）— 25 秒

**画面**: 点击"噪声监测"，展示列表和筛选

**解说词**:

> 噪声监测页面。后端每 5 分钟自动模拟生成数据，管理员也可以手动录入。每条数据入库后，后台自动调用混合阈值引擎进行判断。列表支持按功能区、时间、分贝范围、异常状态多条件筛选。

**操作**:
- 1:35-1:38 点击侧栏"噪声监测"
- 1:38-1:42 鼠标依次指向筛选栏的四个筛选项
- 1:42-1:46 鼠标指向列表中一条标记为"异常"的记录

**画面**: 切换到"告警历史"

**解说词**:

> 被判定异常的自动生成告警。告警类型分三种——超阈值、骤升、夜间异常。管理员可以确认、处置，形成完整的管理闭环。

**操作**:
- 1:46-1:48 点击侧栏"告警历史"
- 1:48-1:52 鼠标指向告警类型标签列（超阈值红色、骤升橙色、夜间异常灰色）

- 1:56-2:00 如果有告警，点击一条的"详情"按钮弹出详情弹窗

> **设计意图**: 噪声+告警合并为一段，快速展示数据流闭环——数据采集→自动判断→告警生成→人工处置。不需要长时间停留。


### ⑤ 统计分析 — ECharts 可视化 + 模型对比（2:00 - 2:35）— 35 秒

**操作**:

- 2:06-2:12 鼠标在时间序列折线图上缓慢划过，展示 tooltip；然后拖一下底部的缩放滑块

**画面**: 切换到"功能区汇总"标签

**解说词**:

**操作**:

- 2:30-2:35 鼠标先在雷达图上划一圈，再在柱状图的"混合模型"蓝色柱上停留

> **设计意图**: 统计页是"证据页"——前面讲了很多"智能化""动态调整"，这里用 ECharts 图表给出硬数据支撑。模型性能对比是收尾前最后一个重头戏。


### ⑥ 系统设置 — ccswitch 实时阈值测试 + 收尾（2:35 - 3:00）— 25 秒

**画面**: 切换到"系统设置"页面

**解说词**:

> 最后看系统设置。这里有一个很能体现"智能化 API"的功能——ccswitch 实时阈值计算测试。我选一个功能区，输入一个分贝值，点测试计算，系统会实时返回：规则阈值是多少、自适应上限下限是多少、混合模式用的是什么、有没有触发异常。整个过程是调用 ccswitch 配置服务的 API 实时计算的，阈值不是死的，是根据当前配置和统计数据动态算出来的。

**操作**:
- 2:35-2:37 点击侧栏"系统设置"
- 2:37-2:40 鼠标指向 ccswitch 配置状态卡片（当前模型、运行时间、配置来源）
- 2:40-2:44 鼠标移到"ccswitch 实时阈值计算测试"卡片，选择"图书馆"，输入 55 分贝
- 2:44-2:47 点击"测试计算"按钮，等待结果展示
- 2:47-2:50 鼠标指向返回的结果行（规则阈值、自适应上限、自适应下限、是否异常、混合模式）

**画面**: 切回仪表盘全景

**解说词**:

> 总结一下。三个核心亮点：第一，混合动态阈值模型，准确率 92.6%，比固定阈值提升近 15 个百分点；第二，系统全自动运行——数据进来自动判断、阈值自动调整、告警自动生成；第三，ccswitch 配置服务支持热更新，阈值参数、AI 模型配置实时生效不用重启。谢谢大家。

**操作**:
- 2:50-2:52 点击侧栏"仪表盘"回到首页
- 2:52-2:58 镜头给仪表盘全景，鼠标缓慢扫过混合引擎面板和四个功能区卡片

> **设计意图**: 用 ccswitch 实时阈值测试做压轴——这是最直观展示"智能化API实时计算"的功能，观众能看到输入一个数字、系统当场算出多个阈值指标。然后快速回到仪表盘做总结，首尾呼应。

## 注意事项

1. **边指边说**: 解说说到哪个 UI 元素，鼠标就同步指向那里
2. **如果你语速偏慢**: 优先压缩第④段（噪声+告警），把时间留给第②③⑤段核心展示
3. **如果某个页面暂无数据**: 直接说"实际运行中这里会展示 XX 数据"，继续往下走
4. **ccswitch 如果没启动**: 系统设置页会显示"未连接"，口播改为"ccswitch 是可选加分项，主系统的所有 P0/P1 功能不依赖它也能正常运行"
5. **如果实时阈值测试接口没返回**: 跳过 2:40-2:50 那 10 秒，直接进总结
6. **录屏工具**: OBS Studio 或 Win+G（Windows 自带 Game Bar）
7. **多录几遍**: 第一遍熟悉节奏，第二遍调整，第三遍正式录。说错不要停，后期可剪
8. **多个组员**: 你负责操作画面，另一个组员负责念稿解说，分工更流畅
