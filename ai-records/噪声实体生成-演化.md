# 噪声实体生成 提示词演化

## v1 — 初始版本

```
帮我生成一个 NoiseRecord 实体类，包含 id、location、decibel、time、deviceId、isAbnormal 字段，用 Java 写。
```

## 问题与不足

v1 产出是一个 POJO，缺少 MyBatis-Plus 注解、表名映射、字段类型约束。`time` 用了 `Date` 而非 `LocalDateTime`，`decibel` 用了 `double` 而非 `BigDecimal`，不符合 CLAUDE.md 规定的精度要求。缺少 `create_time` / `update_time` 审计字段。生成的文件没有 package 声明，不知道放在哪个目录。

## v2 — 改进版本

```
基于 docs/DATABASE_DESIGN.md 中 noise_record 表结构，生成 MyBatis-Plus Entity 类。要求：
- @TableName("noise_record")、@TableId(IdType.AUTO)、@TableField 注解齐全
- 时间字段用 LocalDateTime，分贝值用 BigDecimal(5,1)
- 包含 create_time / update_time / noise_type / noise_duration / judged_by_model 字段
- 放 backend/src/main/java/com/example/noise/entity/ 目录
- 用 Lombok @Data，中文注释每个字段含义
```

## 改进效果

v2 产物正确使用了 MyBatis-Plus 注解体系，字段类型对齐 DATABASE_DESIGN.md 规范，包含了 P0-P2 全量字段。但 `isAbnormal` 字段语义不清（NULL/0/1 三种状态的注释缺失），且没有标注字段对应的优先级（哪些是 P0 必填、哪些是 P2 预留）。

## v3 — 优化版本

```
基于 docs/DATABASE_DESIGN.md §3 noise_record 表 + docs/00-选题标定.md §三 P0 清单，生成 NoiseRecord Entity。

硬约束：
1. MyBatis-Plus 注解：@TableName("noise_record") + @TableId(IdType.AUTO) + 每个字段 @TableField
2. 类型规范：BigDecimal 存分贝（禁止 float/double），LocalDateTime 存时间（禁止 Date）
3. 审计字段：create_time + update_time，MyBatis-Plus 自动填充
4. 注释格式：每个字段标注「字段含义 + 优先级(P0/P1/P2) + 取值范围」
5. isAbnormal 注释必须写清三态：NULL=未判断, 0=正常, 1=异常
6. judgedByModel 注释写清枚举值：RULE_BASED / STATISTICAL / HYBRID
7. 目录：backend/src/main/java/com/example/noise/entity/NoiseRecord.java
8. 代码风格：2空格缩进，Lombok @Data

输出后请在注释中标注每个字段的出处（来自 DATABASE_DESIGN.md 第几行）。
```

## 最终效果

v3 产出的 NoiseRecord.java 在首次生成时即包含全部 10 个字段、三态语义标注、模型枚举注释、优先级标签，无需二次修正。字段映射与数据库脚本 `sql/01-init.sql` 零偏差，Service 层引用时未出现字段名拼写错误或类型不匹配。代码评审 R-05 中 Entity 维度零 issue。
